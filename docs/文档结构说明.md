# 文档文件夹说明

## 📁 文件夹位置

所有项目文档现在都位于 `docs/` 文件夹中。

## 🎯 为什么要整理文档？

1. **项目根目录整洁** - 避免根目录文件过多
2. **便于管理** - 所有文档集中在一个位置
3. **易于查找** - 通过分类和索引快速找到需要的文档
4. **标准化** - 符合常见项目的文档组织结构

## 📂 保留在根目录的文档

以下文档保留在根目录（符合标准做法）：
- `README.md` - 项目主说明文件（GitHub/GitLab等平台会显示）

以下文档保留在原位置（特定模块的文档）：
- `server/README.md` - 服务器端说明
- `server/INSTALL.md` - 服务器安装说明

## 📝 添加新文档

创建新的 `.md` 文件时，请放在 `docs/` 文件夹中：

```bash
# 在 docs/ 文件夹中创建新文档
docs/新功能说明.md
```

如果需要更新文档索引，请编辑 `docs/README.md`。

## 🔍 查找文档

### 按主题查找
- 部署相关 → 部署相关章节
- 数据库相关 → 数据库相关章节
- 问题排查 → 问题排查章节

### 按文件名查找
所有文档文件名都使用中文，描述清楚内容，便于搜索。

## 📋 文档命名规范

建议使用以下格式：
- `功能-描述.md` - 例如：`上传-安全指南.md`
- `问题-解决方案.md` - 例如：`数据库-配置修复.md`
- `操作-说明.md` - 例如：`部署-详细步骤.md`

## 🔗 引用文档

在其他文档中引用时，使用相对路径：
```markdown
详细说明请参考 [部署指南](docs/项目部署详细步骤.md)
```
